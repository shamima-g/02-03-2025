# Screen: Admin Inbox — Message List (Admin)

## Purpose
The protected admin inbox that displays all submitted contact messages, shown after the admin passes the password gate. Read-only; no reply, delete, or mark-as-read actions in the initial scope.

### Empty Inbox Sub-State

When no messages have been submitted yet:

```
+----------------------------------------------------------+
|  Admin Inbox                                             |
|----------------------------------------------------------|
|                                                          |
|     No messages yet.                                     |
|     Submitted contact messages will appear here.         |
|                                                          |
+----------------------------------------------------------+
```

### Loading Sub-State

While the API call to fetch messages is in flight:

```
+----------------------------------------------------------+
|  Admin Inbox                                             |
|----------------------------------------------------------|
|                                                          |
|     (o) Loading messages...                              |
|                                                          |
+----------------------------------------------------------+
```

### API Error Sub-State

If the message fetch fails:

```
+----------------------------------------------------------+
|  Admin Inbox                                             |
|----------------------------------------------------------|
|                                                          |
|  ! Could not load messages. Please refresh the page.     |
|                                                          |
+----------------------------------------------------------+
```

## Elements

| Element | Type | Description |
|---------|------|-------------|
| Page heading | Text (h1) | "Admin Inbox" |
| Message count | Text (optional) | Displays the total number of messages retrieved, e.g., "3 messages". |
| Message card (per message) | Card / list item | One card per ContactMessage (R16, R18). Contains all five fields: name, email, subject, message body, timestamp. |
| Sender name + email | Text | Displayed as "Name <email@example.com>" or on separate lines within the card. |
| Subject | Text | Label + selected subject value, e.g., "Subject: Work Inquiry". |
| Timestamp | Text | ISO datetime or human-readable date/time (e.g., "2026-03-01 14:32 UTC"). Auto-generated by the backend on creation. |
| Message body | Text | Full message text. Long messages display in full (no truncation in scope). |
| Loading indicator | Spinner / text | Shown while the GET /messages API call is in flight. |
| Empty state | Text | Shown when the API returns an empty array. |
| Error state | Alert text | Shown if the API call fails. Prompts refresh. |

## User Actions

- **Read messages:** Admin scrolls through the list and reads each message card.
- **Refresh page:** Re-fetches all messages from the API (no manual refresh button required — browser refresh suffices).

## Navigation

- **From:** Admin Inbox — Password Gate (Screen 5) after successful password entry.
- **To:** No in-app navigation targets from this screen in the initial scope. Admin can navigate away using the browser.

## Notes

- Messages are displayed in the order returned by the API (no client-side sorting in the initial scope) (R19).
- No pagination is implemented (R19).
- No reply, delete, archive, or mark-as-read actions are available (out of scope per FRS).
- The page is only accessible after passing the password gate (R17, BR8). If accessed directly without authentication state, the password gate (Screen 5) should be shown.
- The layout should be readable on all screen sizes (NFR1 — single-column card stack is appropriate).
- Theming follows OS prefers-color-scheme (BR9).
